Routage WAN & OSPF

Interconnexion de Sites & DHCP Relay

WAN OSPF Area 0 DHCP Centralisé IP Helper

Contexte du Projet

L'objectif de ce projet était de concevoir une infrastructure réseau intersites robuste reliant trois capitales européennes : Dublin, Londres et Amsterdam. La contrainte principale était d'assurer une communication fluide via des liaisons séries WAN tout en centralisant l'administration des adresses IP.

J'ai déployé un routage dynamique OSPF (Open Shortest Path First) dans une zone unique (Area 0) pour gérer la topologie maillée. De plus, pour simplifier la gestion, le site de Dublin héberge le serveur DHCP maître, tandis que les routeurs de Londres et Amsterdam agissent comme des agents relais pour distribuer les IP à leurs LANs respectifs.

Architecture & Protocoles

Routage OSPF

Utilisation du protocole à état de lien (Link-State) pour calculer automatiquement les meilleures routes (coût) entre les sites via l'algorithme SPF (Dijkstra).

DHCP Centralisé

Optimisation des ressources : un seul routeur (Dublin) gère les pools d'adresses IP pour l'ensemble des trois sites géographiques.

IP Helper-Address

Configuration des routeurs distants pour intercepter les requêtes DHCP (Broadcast) et les transmettre en Unicast vers le serveur central.

Configuration Cisco IOS

Ci-dessous, la configuration du routeur Dublin (Serveur DHCP & OSPF) et celle du routeur Londres (Client OSPF & Relais DHCP). 

! --- ROUTEUR DUBLIN (Serveur DHCP) ---
ip dhcp excluded-address 172.16.2.1 172.16.2.5
ip dhcp excluded-address 172.16.1.1 172.16.1.5

ip dhcp pool LAN_DUBLIN
 network 172.16.2.0 255.255.255.0
 default-router 172.16.2.1
 dns-server 8.8.8.8

ip dhcp pool LAN_LONDRES
 network 172.16.1.0 255.255.255.0
 default-router 172.16.1.1
 dns-server 8.8.8.8

router ospf 1
 network 172.16.2.0 0.0.0.255 area 0
 network 172.16.3.0 0.0.0.3 area 0
 network 192.168.10.0 0.0.0.3 area 0
 passive-interface GigabitEthernet0/0

! --- ROUTEUR LONDRES (Relais DHCP) ---
interface GigabitEthernet0/0
 description LAN_LONDRES
 ip address 172.16.1.1 255.255.255.0
 ip helper-address 172.16.2.1
 duplex auto
 speed auto

router ospf 1
 router-id 2.2.2.2
 network 172.16.1.0 0.0.0.255 area 0
 network 172.16.3.0 0.0.0.3 area 0
 network 192.168.10.4 0.0.0.3 area 0

Bilan Technique

Ce TP a consolidé ma compréhension du routage dynamique : contrairement au routage statique, OSPF s'adapte automatiquement si une liaison série tombe (convergence). J'ai aussi appris à configurer le DHCP Relay, une fonctionnalité indispensable en entreprise pour éviter de multiplier les serveurs DHCP sur chaque site distant.

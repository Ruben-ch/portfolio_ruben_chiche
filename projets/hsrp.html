Haute Disponibilité HSRP

Redondance de Passerelle & Load Balancing

Cisco IOS Protocole HSRP Routage RIPv2 VLANs 10 & 20

Contexte du Projet

Dans une infrastructure réseau critique, la panne d'un routeur peut isoler tout un segment de l'entreprise. Ce projet vise à implémenter le protocole HSRP (Hot Standby Router Protocol) pour éliminer ce point de défaillance unique (SPOF).

L'architecture met en œuvre deux routeurs physiques (R0 et R1) qui partagent une adresse IP virtuelle (VIP). Les PC utilisent cette VIP comme passerelle par défaut. Si le routeur principal tombe, le routeur de secours prend le relais instantanément, de manière transparente pour les utilisateurs.

Architecture & Fonctionnalités

VIP (Virtual IP)

Les PC ne connaissent qu'une seule adresse de passerelle (ex: 192.168.1.254). C'est une adresse virtuelle flottante entre les deux routeurs.

Load Balancing

Optimisation du trafic : R0 est "Active" pour le VLAN 10, tandis que R1 est "Active" pour le VLAN 20. Les deux liens sont utilisés simultanément.

Failover & Preempt

Grâce à la commande preempt, si le routeur principal revient en ligne après une panne, il reprend automatiquement son rôle de chef.

Configuration Cisco IOS

Voici la configuration des routeurs R0 et R1, sans commentaires, mettant en évidence les commandes de priorité et d'IP virtuelle. 

interface GigabitEthernet0/0/0.10
 encapsulation dot1Q 10
 ip address 192.168.1.1 255.255.255.0
 standby 10 ip 192.168.1.254
 standby 10 priority 110
 standby 10 preempt

interface GigabitEthernet0/0/0.20
 encapsulation dot1Q 20
 ip address 192.168.11.1 255.255.255.0
 standby 20 ip 192.168.11.254
 standby 20 priority 90
 standby 20 preempt

interface GigabitEthernet0/0/0.10
 encapsulation dot1Q 10
 ip address 192.168.1.2 255.255.255.0
 standby 10 ip 192.168.1.254
 standby 10 priority 90
 standby 10 preempt

interface GigabitEthernet0/0/0.20
 encapsulation dot1Q 20
 ip address 192.168.11.2 255.255.255.0
 standby 20 ip 192.168.11.254
 standby 20 priority 110
 standby 20 preempt

router rip
 version 2
 network 192.168.1.0
 network 192.168.11.0
 network 192.168.2.0
 no auto-summary

Bilan Technique

Ce TP m'a permis de comprendre l'interaction entre les VLANs (niveau 2) et la redondance de niveau 3. La configuration des switchs en mode Trunk est essentielle pour laisser passer les trames taguées vers les sous-interfaces des routeurs. J'ai également pu constater que le Load Balancing permettait d'utiliser toute la bande passante disponible plutôt que de laisser un routeur "dormir" inutilement.

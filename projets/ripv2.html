Routage Hybride & IPv6

RIPv2, RIPng, NAT & DHCP

RIPv2 MD5 IPv6 (RIPng) NAT/PAT DHCPv4

Contexte du Projet

Ce projet complexe, réalisé sous Cisco Packet Tracer, vise à concevoir une architecture réseau redondante "Dual Stack" (IPv4 et IPv6). La topologie repose sur trois routeurs (R1, R2, R3) interconnectés en triangle, permettant une continuité de service en cas de rupture d'un lien.

L'objectif principal était de maîtriser la cohabitation des protocoles de routage : RIPv2 (sécurisé par MD5) pour l'adressage classique et RIPng pour le déploiement de l'IPv6. De plus, j'ai implémenté les services essentiels aux utilisateurs : DHCP pour l'attribution automatique des IP et NAT/PAT pour la traduction d'adresses vers le WAN.

Technologies Mises en Œuvre

Sécurité du Routage

Configuration de l'authentification MD5 via Key-Chain sur RIPv2 pour empêcher l'injection de fausses routes dans la table de routage.

Routage IPv6 (RIPng)

Mise en place de l'adressage 2001:db8::/64 et activation du processus RIP Next Generation pour le routage des paquets IPv6.

Services LAN (NAT/DHCP)

Déploiement de serveurs DHCP locaux sur chaque routeur et configuration du NAT Overload (PAT) pour l'accès extérieur.

Configuration Cisco IOS

Voici un extrait des configurations appliquées, montrant l'activation simultanée de IPv4 (avec sécurité) et IPv6 sur les interfaces. 

! --- Configuration IPv6 & RIPng ---
ipv6 unicast-routing
interface GigabitEthernet0/0
 ipv6 address 2001:db8:12::1/64
 ipv6 rip PROCESS_RIPNG enable

ipv6 router rip PROCESS_RIPNG

! --- Configuration IPv4, RIPv2 & Auth MD5 ---
key chain RIP_KEY
 key 1
  key-string cisco123

interface GigabitEthernet0/0
 ip address 10.0.0.1 255.255.255.252
 ip rip authentication mode md5
 ip rip authentication key-chain RIP_KEY

router rip
 version 2
 network 10.0.0.0
 network 172.16.0.0
 no auto-summary

! --- Services DHCP & NAT ---
ip dhcp pool LAN_R1
 network 172.16.10.0 255.255.255.0
 default-router 172.16.10.1

ip nat inside source list 1 interface Serial0/0/0 overload
access-list 1 permit 172.16.10.0 0.0.0.255

Bilan Technique

Ce TP a démontré l'efficacité de la topologie en triangle : la convergence des routes RIP permet une continuité parfaite même si une liaison inter-routeurs est coupée. J'ai acquis une solide compréhension de la syntaxe IPv6 et des différences de configuration avec IPv4 (activation par interface vs déclaration réseau).

Sécurité Réseau & ACL

Filtrage de paquets & Segmentation VLAN

Cisco IOS Sécurité Packet Tracer Matériel Physique

Contexte du TP

Ce projet consistait à sécuriser un réseau d'entreprise segmenté en plusieurs zones (Serveurs, PC fixes, Portables). L'objectif était d'appliquer une politique de sécurité stricte (Least Privilege) en utilisant des Listes de Contrôle d'Accès Étendues (Extended ACLs) sur un routeur Cisco.

Réalisation hybride : J'ai d'abord modélisé l'architecture sur Cisco Packet Tracer pour valider la logique de routage et de filtrage, avant de déployer la configuration en conditions réelles sur du matériel physique Cisco (Switchs 2960 et Routeur ISR) du laboratoire réseau.

Politique de Sécurité Appliquée

Accès Web Restreint

Le réseau des portables n'a accès qu'au serveur Web (172.16.0.1) et uniquement via le protocole sécurisé HTTPS (Port 443).

Protection Inter-VLAN

Isolation stricte : Le portable "invité" (192.168.2.100) est totalement bloqué vers le réseau serveur. Le ping vers les PC fixes est interdit.

Administration SSH

Sécurisation du plan de gestion : Seul le poste Administrateur (192.168.1.1) est autorisé à configurer le routeur via SSH.

Configuration Cisco IOS

Voici les commandes clés injectées dans le routeur. J'ai utilisé des ACLs nommées (Named ACLs) pour une meilleure lisibilité et modularité. 

! --- ACL pour le réseau des PC Fixes (VLAN 10) ---
ip access-list extended ACL_FIXES
 permit tcp 192.168.1.0 0.0.0.255 host 172.16.0.1 eq 443
 permit udp 192.168.1.0 0.0.0.255 172.16.0.0 0.0.255.255 eq 53
 permit tcp 192.168.1.0 0.0.0.255 172.16.0.0 0.0.255.255 eq 53
 deny ip 192.168.1.0 0.0.0.255 172.16.0.0 0.0.255.255 
 deny icmp 192.168.1.0 0.0.0.255 192.168.2.0 0.0.0.255
 permit ip any any
exit

! --- ACL pour le réseau des Portables (VLAN 20) ---
ip access-list extended ACL_PORTABLES
 deny ip host 192.168.2.100 172.16.0.0 0.0.255.255
 deny tcp 192.168.2.0 0.0.0.255 any eq 21
 deny tcp 192.168.2.0 0.0.0.255 any eq 20
 permit tcp 192.168.2.0 0.0.0.255 host 172.16.0.1 eq 443
 permit udp 192.168.2.0 0.0.0.255 172.16.0.0 0.0.255.255 eq 53
 deny ip 192.168.2.0 0.0.0.255 172.16.0.0 0.0.255.255
 deny icmp 192.168.2.0 0.0.0.255 192.168.1.0 0.0.0.255
 permit ip any any
exit

! --- Sécurisation de l'accès Administration (VTY) ---
access-list 10 permit host 192.168.1.1
line vty 0 4
 access-class 10 in
 transport input ssh
exit

! --- Application aux interfaces ---
interface GigabitEthernet0/0
 ip access-group ACL_FIXES in
exit

interface GigabitEthernet0/1
 ip access-group ACL_PORTABLES in
exit

Compétences Acquises

Ce TP m'a permis de maîtriser la syntaxe des ACLs étendues Cisco et de comprendre l'importance de l'ordre des règles (séquentiel). J'ai également appris à diagnostiquer les problèmes de connectivité en analysant où le trafic est bloqué (interface entrante vs sortante). La mise en pratique sur matériel physique a renforcé ma compréhension du câblage et des contraintes réelles par rapport à la simulation.
